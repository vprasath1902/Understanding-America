    EPUB Readiness

    This page documents how the HTML project is structured for eventual EPUB conversion. It does not generate an EPUB by itself, but it gives the book a clean packaging target.

    HTML Source

    Chapters and appendices remain semantic HTML files.

    Shared Assets

    CSS, JavaScript, SVG, and images are kept in stable folders.

    Manifest

    book.json identifies title, author, version, and navigation.

    Export

    Use a future build script or publishing tool to package EPUB.

    EPUB Packaging Notes

    Recommended: keep chapter content free of browser-only dependencies so it can be reused in EPUB and PDF outputs.
    Images: SVG diagrams are ideal for HTML and PDF. EPUB readers vary, so provide PNG fallbacks later for the most important diagrams.
    Navigation: the existing BOOK_NAV array can become the EPUB table of contents during a future build step.
